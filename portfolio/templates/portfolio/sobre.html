Sobre esta Aplicação

1. Arquitetura MVT do Django

A aplicação segue o padrão arquitetural MVT (Model-View-Template) do Django, uma adaptação do clássico MVC.

  • Model: camada de dados que define a estrutura das entidades na base de dados através do ORM do Django. Cada modelo é uma classe Python que herda de models.Model e cada atributo corresponde a uma coluna na tabela.
  • View: camada de lógica de negócio. Recebe pedidos HTTP, interage com os modelos para obter ou modificar dados, e devolve uma resposta — tipicamente um template renderizado.
  • Template: camada de apresentação. Ficheiros HTML com tags do Django Template Language que permitem renderizar dados dinâmicos vindos da View.

Fluxo de informação: O utilizador faz um pedido HTTP → o URL dispatcher identifica a rota → a View correspondente é executada → a View consulta os Models → os dados são passados ao Template → o HTML renderizado é devolvido ao utilizador.

2. Modelação

A aplicação tem 10 entidades principais: Licenciatura, UnidadeCurricular, Docente, Projeto, Tecnologia, TipoTecnologia, TFC, Competencia, Formacao e MakingOf.

As relações utilizam ForeignKey para relações 1:N (por exemplo, uma Licenciatura tem muitas Unidades Curriculares) e ManyToManyField para relações N:M (por exemplo, um Projeto pode usar várias Tecnologias e uma Tecnologia pode estar em vários Projetos).

A entidade MakingOf documenta o processo de desenvolvimento, e a entidade extra TipoTecnologia permite categorizar as tecnologias.

4. Estrutura de Navegação

A aplicação organiza-se em torno de listagens (projetos, tecnologias, TFCs, unidades curriculares, competências, formações), cada uma com operações CRUD acessíveis através de botões. A página Sobre centraliza informação técnica e o Making-Of documenta o processo de desenvolvimento.

5. Repositório GitHub

github.com/luisrdsba/portfolio_project2026

O Git e o GitHub foram fundamentais para o desenvolvimento desta aplicação. Permitiram:

  • Controlo de versões: histórico completo de cada alteração, com possibilidade de reverter qualquer commit
  • Backup na nuvem: o código fica seguro mesmo que algo aconteça ao computador local
  • Trabalho entre máquinas: possibilidade de continuar o desenvolvimento em qualquer computador
  • Colaboração futura: base preparada para trabalhar com outras pessoas no mesmo projeto
